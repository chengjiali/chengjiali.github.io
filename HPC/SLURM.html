SLURM

SLURM workload manager

SLURM stands for Simple Linux Utility Resource Manager.

SLURM commands

Submit a job

Since SLURM manages the resource on the cluster, we have to ask for resources (compute nodes, memory, etc.) from SLURM. You will get the resources from the compute nodes for your job. (In this way you will not end up running computational jobs on the login nodes.) Also we have to tell SLURM what job / command need to be executed.

There are two ways to run a SLURM job:

  • Interactive mode
  • Batch mode.

Interactive mode

“srun” is the command to submit an interactive job for execution. During the interactive job, the ssh connection between you and the cluster has to be maintained. The usage is srun [parameter] [job]

The parapeters controls what resources are allocated and how the tasks are distributed across the nodes. Then plus the job / command to execute.

e.g.
Start a job on your local machine:

The way the srun works is:

  • Allocate resources
  • Log onto the resources
  • Run the job
  • Log out

!! If you want to allocate resources and do not run any job, please do not use salloc. Instead use

#!/bin/bash - Specify the executor of the script

Normal configurations

#SBATCH –job-name=test

#SBATCH –output=test.out

#SBATCH –error=test.err

#SBATCH –time=04:00:00

Parallel resource configs

#SBATCH -p general

#SBATCH -N 2

#SBATCH -n 2

#SBATCH –ntasks-per-node=2

#SBATCH –cpus-per-task=4

#SBATCH –mem-per-cpu=10G

#SBATCH –mem=10G

#SBATCH –gres=gpu:1

Constraint configs

#SBATCH –nodelist=c1[234-238]

#SBATCH –exclude=c1234

#SBATCH –constraint=“E5-2690v3@2.60GHz”

#SBATCH –gres=gpu:k20:1 ## Specify GPU preference

Dependencies configs

#SBATCH –dependency=after:jobid

#SBATCH –dependency=afterok:jobid

#SBATCH –dependency=afternotok:jobid

The jobs you want to run, e.g.
