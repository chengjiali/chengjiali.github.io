Modeling entities and relations

RotatE models entities in complex space and relation as rotation.
Each component of the relation embedding is an angle.

It is expected that applying rotation to head is tail, shown below.

$ t = h .* r $,

where |r_{i}| = 1. With this, we define a distance function for each triplet

$ || d_{r}(h, r) || $.

Specifically, they treat components of relation embeddings as angles.
Then $ e^{i\theta} = sin(\theta)$ and $icos(\theta) $ is
a rotation on the unit circle in complex space.

Basically we scale relation embedding into [-pi, pi]. Each component is an angle.
Compute dot product of a vector and the relation embedding is equivalent to
rotating that vector in complex space.

Calculating score

The score function of RotatE is $ - d_{r}(h, r) $.
score = -|| h . r - t ||^2, where . is element-wise product.
Assume h .* r = t. Let h = a + ib, r = c + id, t = e + if.

When predicting tail:
h . r = t
score = (a + ib) (c + id) - (e + if) = re + i im
re = a c - b d - e
im = a d + b c - f

When predicting head:
Rotate t in reverse direction, i.e. let r = conj(r) = c - id.
score = (c - id) (e + if) - (a + ib) = re + i im
re = c e + d f - a
im = c f - d * e - b
